## 1. 项目愿景

构建一个 **Agent 原生** 的桌面 Word 编辑器，让 AI 深度参与文档创作全流程 —
不只是"对话框里聊天"，而是在编辑器的光标、选区、格式、上下文中与 Agent 自然协作。

**一句话**：像用 Word 一样写文档，像和同事对话一样调用 AI。

**开源**：geex-docx 是开源软件，基于 docx-editor 构建。用户自行配置 LLM API Key 后即可启用 Agent 功能。

## 2. 目标用户与场景

### 2.1 用户画像

| 角色 | 描述 | 核心诉求 |
|------|------|---------|
| **日常办公用户** | 需要频繁撰写报告、方案、邮件 | 格式排版省心、AI 帮忙起草和润色 |
| **重度写作者** | 技术文档、论文、长篇内容创作者 | 大纲展开、段落重写、术语一致性检查 |
| **国际化协作** | 需要中英文文档互译、多语言内容 | 翻译不失排版、术语统一 |
| **开发者/高级用户** | 通过 Agent 接口自动化文档生成 | 可编程、可扩展、可集成工作流 |

### 2.2 核心使用场景

1. **智能起草**：给 Agent 一个标题或大纲，自动生成初稿
2. **交互式润色**：选中一段文字 → 命令面板 → "更正式" / "更简洁" / "翻译成英文"
3. **全文优化**：一键检查拼写、语法、术语一致性、格式规范
4. **排版自动化**：Agent 根据内容类型自动推荐样式（标题层级、列表、表格格式）
5. **文档间联动**：跨文档引用、批量更新、模板套用

## 3. 产品范围定义

### 3.1 MVP (最小可行产品) — v0.1

> 目标：基于 docx-editor 构建可用的桌面 Word 编辑器，集成 pi agent 实现 AI 辅助写作。
> docx-editor 已提供完整的 OOXML 渲染/编辑引擎，MVP 聚焦于封装桌面端能力 + Agent 集成。

**架构**：单页面应用 — WorkspacePage 包含编辑器、大纲、Agent 侧栏、状态栏。Settings 为 Workspace 内的侧边面板。

### 3.2 范围内 (In Scope)

- ✅ 标准 .docx (OOXML) 格式解析、分页渲染、编辑、保存（基于 docx-editor）
- ✅ 完整文本编辑（输入、删除、光标/选区导航、IME 支持）
- ✅ 完整文本格式（加粗/斜体/下划线/删除线/字体/字号/颜色/高亮/上下标）
- ✅ 完整段落格式（对齐/缩进/行距/段间距/边框/底纹）
- ✅ 标题样式 1-6 + 自定义样式
- ✅ 有序/无序列表（多级编号）
- ✅ 表格插入与编辑（由 docx-editor 提供）
- ✅ 图片插入（由 docx-editor 提供）
- ✅ 超链接
- ✅ 页眉/页脚（由 docx-editor 提供）
- ✅ 脚注/尾注（由 docx-editor 提供）
- ✅ 页面设置（页边距、纸张大小、方向）
- ✅ 撤销/重做
- ✅ 查找与替换
- ✅ 缩放控制、标尺
- ✅ 文档大纲导航
- ✅ 模板变量处理
- ✅ 打印（调用系统打印对话框，Should 优先级）
- ✅ Agent 命令面板（选中文本 → AI 处理）
- ✅ Agent 全文处理（校对、润色、翻译、扩写、总结）
- ✅ Agent 批注与修订建议（Track Changes）
- ✅ 自动保存与崩溃恢复
- ✅ API Key 配置（Settings 面板内，写入系统 Keychain）

### 3.3 范围外 (Out of Scope — MVP)

- ❌ 协同编辑 / 实时协作
- ❌ 宏 / VBA 兼容
- ❌ 云存储集成（如 OneDrive / Google Drive）
- ❌ PDF 导出（可在后续版本通过 Tauri 端实现）
- ❌ 内容控件 (Content Controls / SDT) 的 UI 操作（底层已支持）
- ❌ 修订模式（Track Changes）的完整 UI（Agent 可生成建议，但无完整 Track Changes 面板）
- ❌ 许可/激活系统（开源软件，无激活码）
- ❌ 用户注册/登录（开源软件，无账号系统）
- ❌ 社交登录（开源软件，无后端）

## 4. 验收标准

### 4.1 MVP 验收清单

- [ ] 启动即进入 WorkspacePage（无登录/激活页）
- [ ] 能打开 Microsoft Word 创建的 .docx 文件，文本内容正确显示
- [ ] 能在编辑器中输入中文和英文，格式正确
- [ ] 加粗、斜体、下划线、字体、字号、颜色可设置且生效
- [ ] 标题 1-6 样式可应用
- [ ] 有序/无序列表可正常使用
- [ ] 保存后，用 Microsoft Word 打开，格式无丢失
- [ ] 撤销/重做正常工作，至少支持 50 步
- [ ] Cmd/Ctrl+K 能唤起 Agent 命令面板
- [ ] 选中文本后 Agent 能成功润色并替换
- [ ] Settings 面板内可配置 API Key，配置后 Agent 功能正常
- [ ] 应用启动时间 < 3 秒（macOS M 系列芯片）

### 4.2 已知限制（v0.1）

- 不支持 doc 格式（仅 docx / OOXML）
- 不支持图表 (Chart) 的创建和编辑
- 不支持 SmartArt 图形
- 不支持数学公式 (MathEquation) 的 UI 创建（底层模型已支持）
- 不支持内容控件 (Content Controls / SDT) 的 UI 操作（底层模型已支持）
- 不支持修订模式 (Track Changes) 的完整操作面板
- 不支持邮件合并
- 不支持文档保护/加密
- Agent 功能依赖 pi agent 后端 LLM，无 API Key 时不可用（除非配置本地模型）

## 5. 迭代计划

> ⚠️ **版本号体系变更**：本文档定义的 Phase 计划替代 BRD §10 原来的 v0.1.0 → v0.2.0 → v0.3.0 → v1.0.0 版本序列。
> Phase 1-3 合计约等于原计划的 v0.1.0（MVP），Phase 4 约等于 v1.0.0（正式发布）。
> BRD §10 的版本号保留作为发布标签参考，但开发编排以本 Phase 表为准。

| Phase (开发) | 版本 (发布) | 主题 | 核心交付 |
|-------------|------------|------|---------|
| Phase 1 | — | 骨架搭建 | 单页面布局、stores、API Key 配置 |
| Phase 2 | — | 编辑器核心 | DocxEditor 集成、打开/保存、格式化 |
| Phase 3 | v0.1.0-beta | Agent 集成 | 命令面板、对话侧栏、pi 通信 |
| Phase 4 | v1.0.0 | 完善 | 表格/页面布局/审阅/崩溃恢复 |

## 6. 竞品分析

| 产品 | 类型 | 优势 | 不足 |
|------|------|------|------|
| Microsoft Word | 桌面 | 功能全面、格式兼容完美 | 无 Agent 原生能力 |
| Google Docs | Web | 协同编辑强 | 无 Agent、非离线 |
| Notion AI | Web | AI 写作辅助 | 非 docx 格式、非桌面原生 |
| OnlyOffice | 桌面/Web | 开源、格式兼容好 | AI 功能弱 |
