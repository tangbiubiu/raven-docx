产品需求文档：AgentWrite

版本：v0.1 · 状态：草稿 · 作者：产品团队

1. 产品愿景

构建一个 Agent 原生的桌面文档编辑器，让 AI 深度参与文档创作全流程 — 不只是"对话框里聊天"，而是在编辑器的光标、选区、格式、上下文中与 Agent 自然协作。

一句话：像用 Word 一样写文档，像和同事对话一样调用 AI。

2. 核心设计原则

  • 光标即上下文：Agent 始终知道你在文档中的位置和所选内容
  • 内联建议：AI 的建议直接出现在文档中，接受或拒绝一键完成
  • 无感切换：写作和调用 AI 不需要切换界面或上下文
  • 格式保留：所有 AI 操作保留原文档的样式和格式

3. 目标用户

日常办公用户和重度写作者 — 产品经理、技术作者、研究人员、学生，他们每天花数小时在文档工具中，需要写作辅助但不希望跳出写作流。

4. 功能范围

4.1 文档编辑

标准的富文本编辑能力：标题层级、加粗/斜体/下划线/删除线、有序/无序列表、引用块、字体和字号选择。支持 .docx 导入导出（规划中）。

4.2 Agent 协作

  • 上下文感知追问：Agent 知道光标位置和选中的文字
  • 内联续写：从光标位置继续写作，保持语气和格式一致
  • 选区润色：选中文字后一句话让 Agent 改写
  • 全文摘要：一键生成文档摘要（插入光标位置或侧栏展示）
  • 翻译选区：选中文字直接翻译并替换或追加
  • 风格检查：扫描全文给出语气、可读性建议

4.3 协作体验

Agent 像一位坐在你旁边的写作搭档：它看得见你在写什么、选了什么、光标在哪儿；你可以随时对它说一句话，它立刻给出审阅意见或帮你继续写。不需要复制粘贴，不需要切换窗口。

5. 技术方案（草案）

基于 Tauri v2 + Rust 构建跨平台桌面应用。文档内核采用 docx-editor OOXML 渲染/编辑引擎，Agent 通过 Rust 后端 spawn pi agent 子进程（JSONL stdin/stdout 协议）与 LLM 通信。上下文管理模块维护文档状态、光标位置、选区内容和对话历史。

模板变量示例：项目代号为 {项目代号}，当前版本为 {版本号}，负责团队为 {团队名称}。

6. 里程碑

  1. M1 — 基础编辑器：富文本编辑、工具栏、文件打开/保存
  2. M2 — Agent 接入：侧栏对话、上下文感知、快捷操作
  3. M3 — 深度集成：内联建议、选区感知、格式保留
  4. M4 — 协作增强：多人实时协作、评论、修订模式

7. 附录：竞品对比

当前市场上的 AI 写作工具大多以"聊天框 + 文档区"的分离模式存在，用户在两个界面之间频繁切换。AgentWrite 的差异点在于：Agent 是编辑器的一部分，不是外挂。
